# 系统提示工程化 Spec

## 背景
ch04 之后 EasyCode 有了完整的 Agent Loop（ReAct 循环、工具执行、事件流、Plan Mode），能自主干活。但系统提示仍是一个单块长字符串，存在三个结构性问题：

1. 无模块拆分——身份、行为规则、工具使用约束、安全边界全部混杂
2. 无缓存分层——每轮重传稳定内容，Anthropic 的 cache_control 未利用
3. 无法运行时注入——Plan Mode 指令只能靠过滤 tools 实现，空文本重试提示只能伪装成 user 消息

## 目标
把全局指令展开成结构化模块化系统提示，利用 Anthropic cache_control 将稳定内容存入可缓存通道；建立运行时补充指令注入机制。

## 功能需求

- F1: 模块化系统提示拼装——7 个固定模块按 priority 排序装配，3 个可选空槽留待后续，模块间 "\n\n" 分隔
- F2: 环境信息构造——工作目录、平台、日期、Shell、git 状态、版本、模型，作为独立第二段呈现
- F3: 稳定内容走可缓存通道——稳定模块 + tools 定义进缓存，环境信息 + 对话历史不进
- F4: 缓存命中解析——提取 cache_creation_input_tokens / cache_read_input_tokens
- F5: 关键约定双重强化——工具描述和 system prompt 模块中同时出现
- F6: 补充消息注入——<system-reminder> 标签包裹，不写持久历史
- F7: 规划模式按轮次注入——首轮完整，每隔 4 轮重复完整，其余精简
- F8: 跨协议一致——Anthropic 与 OpenAI 行为一致

## 非功能需求
- N1: 缓存确定性——稳定块跨轮逐字节一致
- N2: 不破坏 ch04——所有既有能力不退化
- N3: 历史合法性——注入 reminder 后 role 交替不破坏
- N4: 环境采集不阻塞——git 状态可降级
- N5: 密钥不泄漏
- N6: 缓存字段解析健壮——缺失时值为 0
- N7: 代码规范——mvn spotless:check 通过
- N8: 可扩展性——新增模块不改装配主逻辑

## 不做的事
- 项目指令文件加载、自动记忆、真实 MCP 接入——留待后续
- 历史增量缓存、缓存 TTL 配置化
- 状态栏展示缓存命中
- 提示词运行时热加载
